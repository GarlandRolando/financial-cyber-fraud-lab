# Interview talk track

> I built Financial Cyber-Fraud Lab because digital payments and AI-enabled
> fraud are increasing the need for explainable financial crime analytics.
> The project engineers behavioural features, applies a transparent risk score,
> creates an investigation queue, and evaluates precision and recall using a
> synthetic labelled dataset.
>
> I deliberately started with explainable rules rather than a black-box model.
> In financial services, investigators and regulators need to understand why an
> alert was generated. I also added SQL monitoring queries, validation checks,
> automated tests, and GitHub Actions.
>
> The next version would add graph-based mule-network detection, temporal
> behaviour, model drift monitoring, and investigator feedback.

## Questions to prepare

1. Why use a robust z-score instead of a normal z-score?
2. Why can high recall create operational problems?
3. Why is precision important for investigators?
4. How would you identify mule-account networks?
5. What privacy controls are required?
6. How would you validate thresholds?
7. How can criminals adapt to rule-based detection?
8. Why should an alert not be treated as proof of fraud?
